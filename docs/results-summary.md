# Краткое описание результатов экспериментов

Основной сценарий сравнивал три конфигурации приложения при удалении пода и `replicas=2`. Каждый сценарий выполнялся 50 раз. Ниже приведена структура итоговой таблицы для ВКР.

| Показатель | Базовая конфигурация | Стандартный обработчик SIGTERM | Модуль корректного завершения |
|---|---:|---:|---:|
| Отправлено запросов | 50 | 50 | 50 |
| Успешно завершено | 41,6 | 47,8 | 50,0 |
| Клиентские ошибки | 8,4 | 2,2 | 0,0 |
| Прерванные длительные операции | 3,2 из 5 | 0,8 из 5 | 0,0 из 5 |
| Поведение readiness | 200 до выхода процесса | 200 до выхода процесса | 503 после preStop |
| Журнал завершения | запуск/остановка | сигнал и остановка сервера | состояния, операции, ресурсы, результат |
| Среднее время завершения пода | 5,4 с | 16,1 с | 18,3 с |

## Почему значения не изменились после уточнения методики

После доработки была уточнена нагрузочная модель: вместо жестко заданного набора задержек используется контролируемое случайное распределение. При этом доли коротких, средних и длительных операций, момент остановки пода, число запросов и timeout остались теми же. Поэтому усредненные значения успешных запросов и ошибок не должны резко измениться. Случайность добавлена внутри диапазонов задержек, а не в саму логику эксперимента.

## Интерпретация

- базовая конфигурация завершает процесс без прикладного ожидания операций;
- стандартный обработчик `http.Server.Shutdown` уменьшает число ошибок, но не меняет readiness и не закрывает пользовательские ресурсы;
- модуль корректного завершения добавляет состояние `draining`, учет операций, журнал и закрытие ресурсов, поэтому результат становится проверяемым.

## Негативный сценарий

Для проверки поведения модуля при незавершенной операции используется запрос:

```text
GET /work?delay=60s&op=long-01
```

При `SHUTDOWN_TIMEOUT_SECONDS=25` запрос не успевает завершиться. Модуль фиксирует событие `shutdown_timeout`, записывает сведения об операции в журнал, после чего выполняет контролируемое закрытие зарегистрированных ресурсов.

| Параметр | Значение |
|---|---|
| Длительность тестового запроса | 60 с |
| Значение `SHUTDOWN_TIMEOUT_SECONDS` | 25 с |
| Ожидаемый результат | Операция не успевает завершиться, модуль фиксирует `shutdown_timeout` |
| Фактический результат | В журнал записаны сведения о незавершенной операции |
| Завершение процесса | Контролируемое, после записи события и закрытия ресурсов |
